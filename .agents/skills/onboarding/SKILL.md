# Навичка: Онбординг та налаштування профілю (Onboarding & Profile Setup)

Ця навичка використовується для первинного запуску бази даних та створення профілю користувача.

## Робочий процес

Коли користувач ініціює роботу або просить налаштувати профіль, Агент повинен виконати такі кроки:

### Крок 1. Інтерв'ювання користувача
Поставити користувачеві запитання для заповнення всіх параметрів профілю відповідно до шаблону [templates/profile.md](../../../templates/profile.md) (приклад заповнення: [examples/profile.md](../../../examples/profile.md)):
1. **Ім'я**
2. **Стать**
   > [!IMPORTANT]
   > Стать слід визначити самостійно з контексту спілкування та імені користувача. Якщо однозначно зробити це не вдається — перепитати.
3. **Дата народження** (у форматі `YYYY-MM-DD`)
4. **Місце проживання** (країна, місто)
5. **Чоловік / Дружина / Хлопець / Дівчина / Партнер** (ім'я або вказати, що немає)
   > [!IMPORTANT]
   > Потрібно визначити конкретну роль партнера (чоловік, дружина, хлопець, дівчина або партнер) і записати її в назву параметра профілю (наприклад, замість загального "Чоловік / Дружина / Партнер" написати "Чоловік" або "Хлопець" відповідно до реального статусу). Якщо з відповіді статус незрозумілий — перепитати.
6. **Діти** (імена або вказати, що немає)
7. **Професія** (сфера діяльності / спеціальність)
8. **Робота** (поточна посада або місце роботи)
9. **Захоплення** (хобі та інтереси)

> [!TIP]
> Запитання слід ставити послідовно та дружнім тоном. Допускається згрупувати кілька запитань в один блок (наприклад, особисті дані разом, сімейні разом тощо), щоб не втомлювати користувача.

### Крок 2. Узгодження категорій активностей
Повідомити користувача про базові категорії активностей, які за замовчуванням використовуються для оцінки ментального стану:
* **Сім'я** (спільне дозвілля, спілкування, взаємна підтримка, сімейні конфлікти)
* **Робота** (досягнення, робочі завдання, наради, стрес, перевантаження)
* **Навчання** (отримання нових знань, курси, вебінари, читання професійної літератури)
* **Хобі** (творчість, улюблена справа, відпочинок, захоплення)
* **Саморозвиток** (медитація, рефлексія, психотерапія, особистісні практики)
* **Здоров'я** (сон, харчування, самопочуття, лікування, профілактика)
* **Спорт** (фізичні тренування, розминка, йога, біг, активні заняття)
* **Побут** (прибирання, покупки, приготування їжі, вирішення домашніх справ)

Запропонувати користувачеві:
1. Залишити перелік без змін.
2. Прибрати непотрібні категорії.
3. Додати власні кастомні категорії (наприклад, підготовка до іспитів, специфічні робочі проекти тощо) з коротким описом.

### Крок 3. Створення файлу профілю
На основі отриманих відповідей згенерувати та зберегти файл профілю за шляхом `data/profile.md` відповідно до шаблону [templates/profile.md](../../../templates/profile.md) (орієнтуючись на приклад [examples/profile.md](../../../examples/profile.md)).

Структура файлу має містити:
* Шапку з навігаційними посиланнями: `[**Індекс**](index.md) | [**Журнал активності**](log.md)`
* Таблицю персональних параметрів (з уточненими сімейними ролями).
* Розділ `## Типові категорії активностей` із таблицею категорій (включаючи кастомні, якщо їх було додано).

### Крок 4. Ініціалізація індексу та журналу
1. Створити головний індекс `data/index.md` на базі шаблону [templates/index.md](../../../templates/index.md) (орієнтуючись на приклад [examples/index.md](../../../examples/index.md)).
2. Створити та зробити перший запис у журналі активності `data/log.md` на базі шаблону [templates/log.md](../../../templates/log.md):
   ```markdown
   ## YYYY-MM-DD update | Оновлено: ініціалізовано базу даних та створено профіль користувача <Ім'я>
   - Оновлено: [profile](profile.md)
   - Оновлено: [index](index.md)
   - Оновлено: [log](log.md)
   ```
