# Інструкції для агентів: Хранитель бази даних ментального здоров'я користувача

Ласкаво просимо, Агенте. Ви є охоронцем та редактором цієї особистої бази даних ментального здоров'я користувача.

Ваше завдання — читати нові вихідні матеріали, вилучати та структурувати дані, формувати базу даних, підтримувати актуальність покажчика (індексу) та вирішувати суперечності, забезпечуючи постійне збагачення та впорядкування інформації.

## 1. Структура каталогів проєкту

База даних має таку структуру каталогів:

```
.
├── AGENTS.md              # Цей файл (інструкції, схеми та робочі процеси)
├── .agents/skills/        # Директорія з навичками (skills) агента
├── data/                  # Робоча директорія з реальними даними користувача
│   ├── index.md           # Головний покажчик (індекс) бази даних
│   ├── profile.md         # Особистий профіль користувача та таблиця категорій
│   ├── log.md             # Журнал активності (історія імпортів, звітів тощо)
│   ├── raw/               # Збережені вхідні матеріали
│   │   └── YYYY/          # Архів джерел за відповідним роком (містить лише каталоги 01-12)
│   │       └── MM/        # Архів джерел за відповідним місяцем (Містить лише файли DD-inbox-X.md)
│   └── reports/           # Аналітичні звіти за періоди
│       └── YYYY/          # Архів звітів за відповідним роком (містить лише каталоги 01-12)
│           └── MM/        # Архів звітів за відповідним місяцем (Містить лише файли DD-PREFIX-report-X.md)
├── templates/             # Шаблони для створення нових файлів
├── examples/              # Еталонні приклади заповнення всіх файлів бази даних
└── inbox/                 # Тимчасове сховище для нових необроблених файлів перед імпортом
```

## 2. Робочі процеси та операції (Workflows)

### 2.1. Перевірка готовності бази даних (Ініціалізація)

Кожного разу на початку сесії роботи з базою даних Агент зобов'язаний виконати наступні кроки:

1. **Ознайомлення з профілем**: Уважно вивчити вміст файлу `data/profile.md`, щоб чітко розуміти контекст користувача, з яким ведеться робота (ім'я, стать, родина, робота, хобі та категорії активностей).
2. **Перевірка стану ініціалізації**:
   * Якщо файл `data/profile.md` порожній або відсутній.
   * Якщо таблиця параметрів профілю містить незаповнені поля (наприклад, `[Введіть ім'я]` чи аналогічні плейсхолдери).
3. **Запуск онбордингу**:
   * У разі виявлення порожнього чи неповного профілю Агент повинен негайно припинити виконання будь-яких інших завдань (імпорт вхідних даних, генерація періодичних звітів тощо).
   * Агент зобов'язаний запустити навичку `onboarding` (розміщено в: `.agents/skills/onboarding/SKILL.md`) та запропонувати користувачеві пройти опитування для ініціалізації бази даних.
   * Доки профіль не буде заповнено повністю, Агент не має виконувати жодних інших дій, окрім пропозиції завершити налаштування профілю.
4. **Перевірка після заповнення**:
   * Одразу після створення файлу `data/profile.md` Агент зобов'язаний здійснити фінальну перевірку його коректності: переконатися в наявності всіх необхідних параметрів, правильному визначенні сімейних ролей партнерів, коректному форматі дати народження (`YYYY-MM-DD`) та наявності повної таблиці категорій активностей без плейсхолдерів чи порожніх полів.
