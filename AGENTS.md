# Інструкції для агентів: Хранитель бази даних ментального здоров'я користувача

Ласкаво просимо, Агенте. Ви є охоронцем та редактором цієї особистої бази даних ментального здоров'я користувача.

Ваше завдання — читати нові вихідні матеріали, вилучати та структурувати дані, формувати базу даних, підтримувати актуальність покажчика (індексу) та вирішувати суперечності, забезпечуючи постійне збагачення та впорядкування інформації.

## 1. Структура каталогів проєкту

База даних має таку структуру каталогів:

```
.
├── AGENTS.md              # Цей файл (інструкції, схеми та робочі процеси)
├── .agents/skills/        # Директорія з навичками (skills) агента
├── data/                  # Робоча директорія з реальними даними користувача
│   ├── index.md           # Головний покажчик (індекс) бази даних
│   ├── profile.md         # Особистий профіль користувача та таблиця категорій
│   ├── log.md             # Журнал активності (історія імпортів, звітів тощо)
│   ├── raw/               # Збережені вхідні матеріали
│   │   └── YYYY/          # Архів джерел за відповідним роком (містить лише каталоги 01-12)
│   │       └── MM/        # Архів джерел за відповідним місяцем (Містить лише файли DD-inbox-X.md)
│   └── reports/           # Аналітичні звіти за періоди
│       └── YYYY/          # Архів звітів за відповідним роком (містить лише каталоги 01-12)
│           └── MM/        # Архів звітів за відповідним місяцем (Містить лише файли DD-PREFIX-report-X.md)
├── templates/             # Шаблони для створення нових файлів
├── examples/              # Еталонні приклади заповнення всіх файлів бази даних
└── inbox/                 # Тимчасове сховище для нових необроблених файлів перед імпортом
```
